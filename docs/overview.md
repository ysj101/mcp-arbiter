# MCP Arbiter  
### *A Constitution Layer for Agentic Systems*

---

## 1. 定義

**MCP Arbiter** は、AIエージェントが外部ツール（API・DB・業務システム）を実行する前に、  
その行為の**妥当性・安全性・コンプライアンス**をリアルタイムで審査・制御する  
**ガバナンスレイヤー（Constitution Layer）**です。

従来のエージェントは「できること」を拡張してきましたが、  
MCP Arbiter は「やってよいこと」を定義・制御します。

---

## 2. 背景と課題

### 現状の問題
- エージェントが誤ったツールを実行するリスク
- 機密データの意図しない外部送信
- 不正・過剰な操作（例：誤送信・誤更新）
- ツール連携（MCP）の増加による**統制の不在**

### 本質的な課題
> 「AIが何をできるか」ではなく  
> **「AIに何をさせてよいか」を定義できていない**

---

## 3. ソリューション

MCP Arbiter は、エージェントのツール実行を以下の3段階で制御します：

### ① Intent解析
- エージェントの行動意図を構造化
- 実行対象・対象データ・影響範囲を抽出

### ② Policy評価
- 事前定義されたポリシーに照合
- リスク・コンプライアンス・権限を評価

### ③ 判定・制御
- 実行可否を決定
  - ✅ 許可（Allow）
  - ⚠️ 要承認（Escalate）
  - ❌ 拒否（Deny）
- 理由を説明可能な形で返却

---

## 4. 特徴

### 🔐 リアルタイム制御
ツール実行前に必ず介入し、事故を未然に防止

### 🧠 意図ベース判断
単純なルールではなく、文脈・目的を考慮

### 🧾 説明可能性
すべての判断に理由を付与（監査対応）

### 🔌 MCPネイティブ設計
あらゆるツール連携に横断的に適用可能

### 🏢 エンタープライズ対応
権限管理・監査ログ・ポリシー運用を前提設計

---

## 5. アーキテクチャ概要

```
[User / System]
        ↓
[Agent (Planner)]
        ↓
[MCP Arbiter]
   ├─ Intent Analyzer
   ├─ Policy Engine
   └─ Decision Engine
        ↓
[Tool / API / DB]
```

---

## 6. ユースケース

### 📧 メール送信制御
- 社外ドメインへの送信 → 要承認
- 機密情報含有 → 自動拒否

### 🗄️ データ操作
- 本番DB更新 → 承認必須
- 大量削除 → ブロック

### 💼 業務システム連携
- ERP操作 → 権限チェック
- 財務データ取得 → ロール制御

---

## 7. 従来手法との比較

| 観点 | 従来のAI | MCP Arbiter |
|------|--------|------------|
| 制御タイミング | 実行後 | 実行前 |
| 判断基準 | ルールベース | 意図＋ポリシー |
| 説明可能性 | 低い | 高い |
| セキュリティ | 脆弱 | 強固 |
| エンタープライズ適用 | 困難 | 容易 |
